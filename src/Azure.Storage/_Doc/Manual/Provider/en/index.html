BlobStorageController

BlobStorageController is a controller for connecting to Azure Blob Storage and performing file operations.
The following items can be specified in the JSON options. Specify each item using JSON syntax with the appropriate key and value.

Type Name

GeoLibrary.ORiN3.Provider.Azure.Storage.O3Object.Controller.BlobStorageController, GeoLibrary.ORiN3.Provider.Azure.Storage

JSON Options

Key Type Required Encrypted Description Value Range Default
Account Name string ✓ The Azure Storage account name to connect to. Follows Azure naming rules
Account Key string ✓ ✓ The access key for the Azure Storage account. Valid access key
Use Https bool Whether to use HTTPS `true` or `false` true
Endpoint Suffix string The endpoint suffix for Azure Any suffix core.windows.net
Proxy Uri string Address of the proxy server Valid URI

Parent Object

Root

Child Objects

UploadBlockFromDirectory

Uploads all files in a local directory to Azure Blob Storage as Block Blobs.
Files in subdirectories are not included.
If an error occurs during processing, subsequent uploads are aborted.
The number of successfully uploaded files can be checked using the "Uploaded Count" return value.
The blob names will match the local file names.
Directory names in the local path are not used, so specify a Prefix if necessary.

BlockBlobFile

BlockBlobFile is a class for handling Block Blobs in Azure Blob Storage.
It mainly performs the following operations:

  • Connects using Azure Storage account name and access key
  • Downloads the specified blob file and stores it locally
  • Allows reading of the downloaded blob file
This object is read-only and cannot be used to write data.
The following items can be specified in the JSON options. Specify each item using JSON syntax with the appropriate key and value.

Type Name

GeoLibrary.ORiN3.Provider.Azure.Storage.O3Object.File.BlockBlobFile, GeoLibrary.ORiN3.Provider.Azure.Storage

JSON Options

KeyTypeRequiredDescriptionValue RangeDefault

PageBlobFile

PageBlobFile is a class for handling Page Blobs in Azure Blob Storage.
It mainly performs the following operations:

  • Connects using Azure Storage account name and access key
  • Directly accesses the specified blob file for reading and writing
You can also create a new Page Blob if necessary, but the blob size must be a multiple of 512.
The 'Length' parameter is only used when creating a new file.
The following items can be specified in the JSON options. Specify each item using JSON syntax with the appropriate key and value.

Result

  • Unknown (-1): Unknown error
  • Success (0): Operation completed successfully
  • AzureError (1): Error reported from Azure SDK or service
  • OtherError (2): Other errors (network issues, unexpected errors)
